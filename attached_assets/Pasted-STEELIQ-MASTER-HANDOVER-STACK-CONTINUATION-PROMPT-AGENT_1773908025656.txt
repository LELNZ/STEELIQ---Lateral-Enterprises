==================================================
DELIVERABLE 2 — LABOUR MODEL REDESIGN
==================================================

Current problem:
- labour is effectively too flat
- one labour row per process is applied once per config/task row, then multiplied by item quantity
- it does not scale properly by:
  - member count
  - holes
  - slots
  - mullion/transom ends
  - screws
  - glue joints
  - pane count / pane size

That is not commercially accurate enough.

Required redesign:
Use a hybrid setup-plus-driver labour model.

Formula:
Total Labour Minutes
=
Setup Minutes
+
(Driver Quantity × Minutes Per Driver)

This phase should be per item, not batched across multiple identical units.

==================================================
APPROVED DRIVER LOGIC FOR V1
==================================================

Use these manufacturing labour drivers:

1. CNC Cutting
- setupMinutes = 5 minutes per item/run
- driverType = per_cut_cycle
- minutesPerDriver = 1 minute
- important: machine cuts two ends in one cycle, so the driver should be based on member count / cut cycles, not raw end count

2. Drilling
- outer frame corners only
- 4 holes per outer corner
- driverType = per_hole
- minutesPerDriver = 1 minute
- do NOT apply drilling time to mullion/transom joints in this phase

3. Slotting
- driverType = per_slot
- minutesPerDriver = 5 minutes
- minimum two slots where that process is required

4. Milling
- driverType = per_end
- minutesPerDriver = 5 minutes
- applies to each mullion/transom end

5. Assembly Screwing
- driverType = per_screw
- minutesPerDriver = 0.5 minutes (30 sec)

6. Glue
- 1 minute per outer frame corner
- 1 minute per mullion/transom joint/end
- may be represented as:
  - per_corner
  - per_joint
or another safe equivalent, but the final calculated logic must reflect both outer corners and mullion/transom joints

7. Glazing
Use pane area bands:
- small = 10 min
- medium = 15 min
- large = 20 min
- extra large = 25 min

These pane-size thresholds must be configurable in the library/admin surface if safely possible in this phase.
If not safely possible, implement sensible defaults and report the limitation honestly.
