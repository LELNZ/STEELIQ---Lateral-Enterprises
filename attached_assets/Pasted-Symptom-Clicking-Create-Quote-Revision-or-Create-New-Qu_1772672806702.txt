Symptom

Clicking “Create Quote (Revision)” or “Create New Quote Number” fails with:

getGlassRValue is not defined

Yet the Item Specifics panel shows an R-Value (e.g. 0.37), so the UI layer has access to the logic, but the exec-summary quote generation path does not.

Likely root cause (high confidence)

In client/src/pages/exec-summary.tsx (or a helper it calls), code references getGlassRValue(...) but:

it’s not imported, or

it’s imported with the wrong name/casing (e.g. getglassrvalue, getGlassRvalue), or

it was moved/renamed and the exec-summary code still calls the old symbol.

Because the error is “not defined” (not “is not a function”), it’s almost certainly a missing identifier at runtime, not bad input.

The R-value badge in the builder uses that function successfully (so the function exists in @shared/glass-library).

The crash only happens when generating the quote from exec summary.
