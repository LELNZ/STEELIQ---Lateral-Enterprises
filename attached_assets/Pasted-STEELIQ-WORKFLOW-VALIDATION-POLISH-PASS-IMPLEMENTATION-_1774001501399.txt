PHASE B - LIFECYCLE MEASUREMENT LOGIC
==================================================

OBJECTIVE
Replace the rigid “Site Measure / Review” stage with configurable real-world workflow logic.

BUSINESS REQUIREMENT
Measurement can be:
1. required before estimate/quote
2. required after quote acceptance for final verification
3. not required at all for SED / supply-only / confirmed-drawing jobs

IMPLEMENT
1. Add measurement requirement model
- Add a field for measurement requirement with values:
  - pre_quote_required
  - post_acceptance_required
  - not_required

2. Add dimension source field
- Add a field for dimension source with values such as:
  - site_measure
  - confirmed_drawings
  - client_supplied_dimensions
  - engineer_drawings
  - architectural_drawings
  - other

3. Lifecycle rendering rules
- If pre_quote_required:
  show measurement stage before estimate completion
- If post_acceptance_required:
  show measurement stage after acceptance and before downstream production/install
- If not_required:
  hide measurement stage entirely

4. Dynamic stage labeling
- pre_quote_required -> “Pre-Quote Site Measure”
- post_acceptance_required -> “Final Site Verification”
- not_required -> hidden

5. Defaults
Where sensible, set default behavior:
- renovation -> pre_quote_required
- supply-only / confirmed drawings -> not_required
- install-sensitive / final-manufacture verification -> post_acceptance_required

ACCEPTANCE CRITERIA
- Lifecycle behaves correctly in all three measurement scenarios.
- Irrelevant measurement stage is hidden.
- User can understand why measurement is or is not required.
