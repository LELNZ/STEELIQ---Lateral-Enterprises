1. Customer photos are not being rendered in the quote preview

In the current build, the quote document model includes item photos, but quote-preview.tsx does not render item.photos anywhere.

So even if the photo is saved correctly, the actual quote preview will still not show it.

That part is a confirmed UI/rendering gap.

2. The generated drawing image is likely failing for a different reason

The preview page does try to render the generated drawing using:

/api/drawing-images/:key

So if you are seeing a broken image when clicking it, that usually means one of these is happening:

the drawingImageKey is saved in the quote snapshot, but the PNG file is missing on disk

the drawingImageKey is empty / stale / invalid for that revision

the image route is returning 404 or invalid data

the quote item list is referencing the image key correctly, but the file no longer exists

So for the generated image, this looks more like a media integrity / file availability problem, not just a rendering omission.
