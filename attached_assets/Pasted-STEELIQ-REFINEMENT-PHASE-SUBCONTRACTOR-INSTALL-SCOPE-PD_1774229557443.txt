CURRENT LIVE / REVIEW TRUTH

The subcontractor install scope PDF is conceptually good, but the current output is too heavy and too repetitive for real subcontractor use.

Current problems identified from live review:
1. Cover page is unnecessary as a standalone page and should be merged into page one.
2. “Making Good” is ambiguous and may be poorly interpreted.
3. There are effectively multiple schedule layers / duplicated schedule content.
4. Detailed item pages are too heavy because they are currently one item per page.
5. Site access / variation notes section does not clearly say who is filling it out or how it is used.
6. Pricing Return section B overlaps with the site/access/variation section.
7. Pricing Return section C duplicates the item schedule instead of reusing it.
8. Overall document is too long/heavy and should be reduced substantially.

PRIMARY OBJECTIVE

Refine the subcontractor PDF into a leaner, clearer, more professional issue pack that:
- is easier for subcontractors to read and price from
- avoids duplicated schedules/sections
- uses fewer pages
- retains project-specific value
- keeps the export operationally useful without exposing internal commercial data

PHASE A — MERGE / SIMPLIFY THE FRONT OF DOCUMENT

1. Remove the standalone cover page format.
2. Merge the useful cover-page content into page one together with the most important scope summary information.
3. Page one should include, in a compact and branded layout:
- project / estimate name
- address
- date issued
- prepared by
- scope type (Renovation / New Build)
- work package (Install only / Removal + disposal + install)
- item count
- short issue-purpose statement
- concise scope matrix / summary

4. Add SteelIQ / company branding appropriately so page one looks like a professional issue document.

PHASE B — CLARIFY WORDING

1. Replace or improve ambiguous wording like “Making Good”.
2. Use clearer construction wording such as:
- “Making good to plaster / trims / cladding / finishes”
- or equivalent repo-consistent wording that clearly indicates repair/finishing responsibility

3. Review the PDF wording for:
- inclusions
- exclusions
- assumptions
- potential variations

Make sure it reads clearly for subcontractors and avoids ambiguous trade-language where possible.

PHASE C — REMOVE DUPLICATE SCHEDULE STRUCTURES

1. Consolidate the multiple schedule-like sections.
2. The PDF should have one primary item schedule that serves as the main issue schedule.
3. Do not repeat essentially the same item list again later unless there is a strong reason.

Preferred structure:
- one main schedule table early in the document
- one final pricing return summary page
- no redundant second/third schedule if the first one can be designed properly

PHASE D — REWORK DETAILED ITEM PAGES INTO COMPACT GRID FORMAT

1. Replace the one-item-per-page detailed drawing layout with a compact multi-item page layout.
2. Target a practical density such as:
- approximately 6 to 10 items per page, depending on readability
- not one item per page

3. Each grid cell / block should include at minimum:
- item ID
- small drawing
- dimensions
- qty
- optionally location / layout if space allows

4. Keep the drawings readable, but prioritize reducing total page count and PDF weight.

5. If item photos or site photos are included, group them sensibly rather than bloating every item page.

PHASE E — SITE ACCESS / VARIATION NOTES CLARITY

1. Clarify who this section is for and how it should be used.
2. Reframe it so it reads as one of:
- pricing assumptions for subcontractor review
- subcontractor notes / confirmations
- variation-prone site conditions to review before pricing

3. Remove the ambiguity around who is filling out the checklist.
4. If necessary, merge this content into the pricing return / assumptions section instead of keeping it as a separate standalone page.

PHASE F — MERGE PRICING RETURN WITH MASTER SCHEDULE LOGIC

1. Remove unnecessary duplication between:
- site/access/variation notes
- pricing return assumptions/exclusions/variation section
- item-by-item pricing table

2. Preferred outcome:
- one main master schedule that can be referenced for pricing
- one final pricing return section with:
  - total price
  - GST
  - lead time
  - assumptions
  - exclusions
  - variation notes
  - optional item-by-item pricing columns only if they are not duplicating the schedule wastefully

3. If practical, the main item schedule should be usable as the pricing schedule too.
4. Reduce total duplication substantially.

PHASE G — REDUCE PDF WEIGHT / PAGE COUNT

1. Optimize the document so it is materially leaner than the current version.
2. Reduce unnecessary page count.
3. Keep image quality sufficient for subcontractor reading/printing, but avoid over-heavy output.
4. Report approximate before/after page count and any file-size improvement if measurable.
