Refinement 5 — T007 wind zone rule should track whether it was auto-set

Right now you say:

only auto-set if windZone is currently empty

That’s safe, but slightly limited. If the user started with auto-set value and then frame type changes again, you may want the rule to keep helping unless the user manually overrode it.

For Monday, your current logic is okay.
But a slightly better version is:

auto-fill if wind zone is empty

or if wind zone still equals the last auto-filled value

That prevents “sticky incorrect default” after frame-type changes.

Suggested wording

Auto-fill windZone when it is empty, or when it still matches the previously auto-filled wind zone. Do not override a user-edited wind zone.

Small implementation caution

Your T005 says:

apply preset at form.reset() time (when adding new item or canceling edit)

If the user is editing an item and hits cancel, do you want:

That’s an important safeguard.
