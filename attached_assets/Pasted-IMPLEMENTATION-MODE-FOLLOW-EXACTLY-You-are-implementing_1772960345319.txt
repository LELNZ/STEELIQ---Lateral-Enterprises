--------------------------------------------------
T001 — Populate totalValue on quote creation and revision update
--------------------------------------------------

Problem:
The quotes table contains a `totalValue` column but it is never populated.

Solution:
Populate `totalValue` from the quote snapshot totals.

Implementation:

1. In `server/routes.ts`
   - When creating a quote (`POST /api/quotes`):
     • Extract the sell value from the snapshot JSON.
     • Use either:
       snapshot.totals.sell
       OR
       snapshot.totalsBreakdown.totalInclGst (if that is the canonical total)

2. Store that value into `quotes.totalValue`.

3. When creating a new revision of an existing quote:
   - Update the parent quote's `totalValue` to the new revision's sell value.

Acceptance Criteria:

• Quotes list Value column displays numbers instead of "—"
• Sorting by Value works correctly
• Existing behavior of revisions remains unchanged

T004 — Backfill totalValue for existing quotes
--------------------------------------------------

Problem:
Existing quotes already in the database have `totalValue = NULL`.

Solution:
Backfill values from the latest revision snapshot.

Implementation:

For each quote where `totalValue IS NULL`:

1. Find the current/latest revision.
