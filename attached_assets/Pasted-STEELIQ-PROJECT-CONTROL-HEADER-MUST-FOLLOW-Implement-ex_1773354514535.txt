STEELIQ PLATFORM PROFILE

Purpose
SteelIQ is the shared business operating system for Lateral Enterprises, starting with estimating and quoting, then extending into jobs, invoicing, customer management, production, and division-specific manufacturing modules.

Vision
Build a modular, enterprise-grade, multi-division platform for:
- Lateral Engineering
- Lateral Joinery
- Lateral Laser Cutting

System positioning
- GoHighLevel remains the CRM front office for leads, pipeline, marketing automation, and follow-up workflows
- SteelIQ is the commercial ERP and operational core for customers, projects, estimating, quoting, acceptance, jobs, invoicing, cost tracking, and division-specific operational modules
- Xero remains the accounting source of truth for invoices, payments, and financial ledger data

Direction
Build in controlled layers:
1. Quote engine
2. Identity and access
3. Customers and projects
4. Acceptance and job conversion
5. Invoicing and Xero
6. Customer portal
7. Division-specific production modules

Architecture principles
- Preserve QuoteDocument -> QuoteRenderModel -> Preview/PDF
- Shared core, division-specific operations
- Freeze issued and accepted commercial history
- Convert accepted quotes into jobs without mutating historical quote data
- Integrate with Xero rather than replacing accounting
- Replace spreadsheet logic module by module, not all at once
- Keep modules logical, staged, and reusable

MODULE CONTRACT RULE

Every new or updated module must define and preserve:
- its input contract
- its owned editable state
- its downstream output contract

Do not let modules mutate upstream frozen commercial records.

Examples:
- Jobs consume accepted quote revisions but do not mutate accepted quote history
- Invoices consume accepted/job state but do not rewrite quote history
- Production modules consume job/operational records, not raw quote-builder state
- PDF output consumes QuoteRenderModel and does not invent alternative document state

IMPLEMENT ONLY THE FOLLOWING

1. Image compression in PDF generation
Review all image insertion paths used by the PDF engine.

Requirements:
- stop embedding unnecessarily large lossless images when not needed
- where appropriate, convert image/canvas exports from PNG to compressed JPEG before embedding
- compress drawings, uploaded images, logo rendering, and preview-generated canvases where applicable
- keep visual quality professional and readable
- do not degrade images so badly that the quote looks poor

Preferred approach:
- use JPEG for photographic or large raster content where practical
- keep only those assets as PNG that genuinely require transparency or sharp line quality and do not explode size
- be practical, not dogmatic

2. Image resizing before embedding
Before inserting images into the PDF:
- clamp maximum image dimensions
- resize/downsample oversized source images/canvases before embedding

Use sensible limits for:
- logo
- item drawings
- photos/images

Do not embed original full-resolution assets if the rendered display size is much smaller.

3. Enable PDF compression
If jsPDF supports compression in the current implementation path, enable it.
Use the safest supported compression mode consistent with the existing codebase.

4. Fix rich-text vertical layout / block spacing
Review the rich-text PDF rendering for:
- headings
- paragraphs
- bullets
- numbered lists
- spacer blocks
- bold-heading style blocks used by Additional Capabilities

Requirements:
- each block must calculate/use a reliable height
- ensure page space before rendering each block
- advance Y position correctly after each block
- do not rely on loose estimates that cause overlap
- maintain readable spacing between legal/commercial text groups

5. Page-break safety for rich text
Ensure rich-text sections use page-break protection consistently.
If remaining space is insufficient for the next block, add a new page before rendering that block.

6. Preserve existing quote design
Do NOT redesign:
- header
- quote summary
- schedule layout
- acceptance layout
- job conversion UI
- customer/project UI

This is a rendering-stability phase, not a redesign phase.
