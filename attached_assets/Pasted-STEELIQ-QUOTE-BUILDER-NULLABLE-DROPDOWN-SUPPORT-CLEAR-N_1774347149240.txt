--------------------------------------------------

PROBLEM

In Quote Builder, dropdown fields (e.g. Handle, Lock) cannot be cleared once selected.

This causes:
- invalid configurations (e.g. fixed window with handle)
- incorrect pricing
- poor UX

--------------------------------------------------

OBJECTIVE

Implement a consistent system-wide pattern for nullable dropdowns.

Users must be able to:
- select a value
- revert back to “no selection”

--------------------------------------------------

PHASE A — IDENTIFY TARGET FIELDS

Audit Quote Builder and identify all dropdown/select inputs where:
- the field is optional
- a null/empty state is valid

At minimum include:
- Handle
- Lock
- Hardware-related dropdowns

Do not assume — use repo truth.

--------------------------------------------------

PHASE B — IMPLEMENT “NONE” OPTION

For each applicable dropdown:

Add a first option:
- label: "None"
- value: null (or existing null-equivalent if already standardized)

Rules:
- Must integrate cleanly with existing Select component
- Must not break controlled input behavior
- Must not require schema change if null is already supported

--------------------------------------------------

PHASE C — NORMALIZE EXISTING INCONSISTENCIES

If any dropdown uses:
- "__none__"
- "N/A"
- empty string

Normalize behavior so UI consistently shows:
→ "None"

But:
- do not migrate historical data
- only normalize UI + input handling

--------------------------------------------------

PHASE D — STATE HANDLING

Ensure:

Selecting "None":
- sets field value to null
- updates UI immediately
- removes any dependent pricing contribution if applicable

Do NOT change pricing logic — only ensure null flows through correctly.

--------------------------------------------------

PHASE E — DEPENDENCY SAFETY (CRITICAL)

Where repo truth supports product-type logic:

If configuration changes make a field invalid:

- changing to Fixed Window

- automatically clear that field (set to null)

Only implement where safe and already logically supported.
