PART 2 — T024 QUOTE LIFECYCLE MANAGEMENT
==================================================

Goal:
Add a proper quote lifecycle so quotes are linked to estimates and can be archived/deleted safely.

Business problem:
The user deleted test estimates and the old quotes were left behind.
We need proper relationship handling and user prompts.

Required quote lifecycle model:
Each quote should support lifecycle state fields such as:

- quoteId
- estimateId
- tenantId
- divisionId
- quoteType
- status
- createdAt
- updatedAt
- archivedAt
- deletedAt

Status values:
- draft
- generated
- archived
- deleted

If the codebase already has similar fields/types, extend them carefully instead of duplicating concepts.

Required relationship rule:
Quotes should be treated as linked/derived documents from an estimate, not as disposable temporary UI artifacts.

Implement cascade handling:

A. When an estimate is archived:
Prompt the user:
- Archive estimate only
- Archive estimate + linked quotes

Default action:
- Archive estimate + linked quotes

B. When an estimate is deleted:
If linked quotes exist, prompt the user:
- Delete estimate only
- Delete estimate + archive linked quotes
- Delete estimate + permanently delete linked quotes

Default action:
- Delete estimate + archive linked quotes

C. If a quote is intentionally kept after estimate deletion:
Keep the quote, but mark it clearly as orphaned:
- "Estimate removed"
or equivalent badge/text

Important:
- Do NOT silently hard-delete linked quotes by default
- Default to archive when possible
- Hard delete must be explicit

Implementation notes:
- Add or update quote data model/types
- Add service/helper functions for:
  - getQuotesByEstimateId
  - archiveQuote
  - deleteQuote
  - archiveQuotesByEstimateId
  - deleteQuotesByEstimateId
  - detectOrphanedQuotes
- Add UI confirmation modal/dialog logic where estimate archive/delete actions occur
- Keep implementation modular so it remains compatible with future QuoteDocument/Renderer work
- If there is no audit/event log yet, structure code cleanly so logging can be added later
