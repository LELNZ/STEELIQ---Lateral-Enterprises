Change 2 — be careful with mutation data

This part needs tightening:

For the spec-display save mutation, use doc.metadata.quoteId and doc.metadata.revisionId

That is reasonable only if those IDs are truly part of the document model contract.

I would define this more carefully:

use doc.metadata.quoteId only if QuoteDocumentModel is explicitly the source of quote identity for preview actions

use doc.metadata.revisionId only if revisions are already real and stable in the API/model

if revision identity is not actually first-class yet, do not invent it just to satisfy the refactor

So I’d rewrite that requirement as:

Use QuoteDocumentModel metadata for preview-related persistence actions only where those identifiers are already part of the documented model contract. Do not introduce speculative metadata fields just to remove a local variable.

That avoids forcing premature model shape.
