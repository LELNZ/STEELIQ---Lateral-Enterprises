3. ROOT CAUSE / UX ISSUE ADDRESSED PER PHASE
==================================================

PHASE A — + New Menu Honesty
  Root cause: The global quick-create menu offered "New Project", "New Job", "New Invoice"
  as one-click actions. In the actual workflow, Projects come from accepted quotes,
  Jobs come from Project context, Invoices from billing context — none are valid
  standalone-create actions from anywhere in the app. The menu was lying about workflow.
  
  Fix: Removed New Project, New Job, New Invoice from the dropdown.
  Kept: New Estimate, New Customer, New Contact — all genuinely valid from anywhere.
  File: client/src/App.tsx (dropdownItems array)

PHASE B — Quote Builder Desktop Polish
  Root cause: The items panel heading was unlabelled (no "Quote Schedule" title),
  the collapse/expand button was icon-only with no text label (unclear affordance),
  the Save button had no visual separation from preceding utility controls (read as same
  priority as camera/settings buttons), and the panel itself lacked border treatment
  making it visually merge with the drawing area.
  
  Fixes:
  - Items panel heading: "Quote Schedule" with border-b bg-muted/30 background
  - Secondary metrics (item count, m², excl. price) shown as muted text in heading bar
  - Collapse/Expand button shows text label ("Collapse" / "Expand") with chevron icon
  - Visual separator (w-px h-5 bg-border mx-0.5) added before Save button in action bar
  File: client/src/pages/quote-builder.tsx (lines ~3301–3321 and ~1639)

PHASE C — Invoice Unlinked-State Clarity
  Root cause: Legacy invoices created before the workflow hardening (before customer/project
  linkage was enforced) appeared in the list with blank customer and project cells.
  Blank cells are indistinguishable from a render delay or data error — the list silently
  looked broken with no explanation or repair path.
  
  Fixes:
  - Missing customer: amber Badge "Unlinked" + "Link customer →" link to the source quote
  - Missing project: italic "No project" text + "Review record →" link to the source quote
  - Correctly linked rows remain clean with no added indicators
  File: client/src/pages/invoices.tsx (customer and project cell rendering)

PHASE D — Settings/Users Shell Standardization
  Root cause: The Users (/admin) page still used old floating card layout with no
  icon badge, no subtitle, and no sticky header — visually inconsistent with all
  improved list pages. Settings already had the library shell.
  
  Fix: Applied library benchmark shell to Users page:
  - sticky header: UsersIcon badge, "User Management" title, "Manage system users…" subtitle
  - Right-side header actions: "Role Guide" (info dialog) + "Add User" button
  - Body: flex-1 overflow-auto with max-w-5xl mx-auto content constraint
  File: client/src/pages/users.tsx

PHASE E — Enterprise Consistency Pass
  Root cause: The Quotes list (/quotes) still used the old p-4 sm:p-6 floating layout
  with an inline h1 and icon — same "older generation" feel as other pages pre-Phase 1.
  
  Fix: Applied library shell to quotes-list.tsx:
  - Outer: flex flex-col h-full bg-background
  - Header: sticky border-b bg-card with BookOpen icon badge, "Quotes" title,
    "All formal quotes across divisions" subtitle
  - Body: flex-1 overflow-auto p-4 sm:p-6
  File: client/src/pages/quotes-list.tsx

  Confirmed all other touched pages already had library shell:
  projects-list, jobs-list, op-jobs-list, invoices, customers, contacts, library, settings
