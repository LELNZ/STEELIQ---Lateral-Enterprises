T049 — Establish the company-wide default quote composition
--------------------------------------------------

Problem:
The system now has a shared master template and aligned preview/PDF rendering, but the quote still needs a stronger company-standard document structure and visual hierarchy.

Required implementation:

1. Refine the company-wide default quote composition so it follows this structure as closely as practical:

PAGE 1 PRIORITY ORDER
- header / branding
- quote title + key metadata
- customer + project details
- quote summary / totals
- concise exclusions / terms intro if appropriate

FOLLOWING PAGES
- schedule of items
- drawings and related site photos within item cards

FINAL PAGE / FINAL SECTION
- payment terms / remittance details
- acceptance block

2. Keep this as the default company-wide quote structure unless there is a strong reason not to.

3. Do not force fake page behavior into preview if it adds complexity. The preview can remain a scroll document, but the visual hierarchy should reflect this intended document order.

Acceptance criteria:

- acceptance/payment content sits at the end rather than interrupting the main flow
