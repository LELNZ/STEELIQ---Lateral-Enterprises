PHASE A — REMOVE MISLEADING GLOBAL QUICK CREATE
==================================================

OBJECTIVE
Make the + New menu truthful and aligned to actual workflow dependencies.

CURRENT ISSUE
The current + New menu shows:
- New Estimate
- New Project
- New Job
- New Invoice

But in the actual workflow:
- Estimate can be created directly
- Project normally comes from accepted quote / quote context
- Job normally comes from accepted quote with project linkage
- Invoice normally comes from accepted quote / billing context

This makes the control misleading.

REQUIRED IMPLEMENTATION
1. Update the global + New menu so it only contains truly valid global-create actions.
Required kept actions:
- New Estimate
- New Customer
- New Contact

2. Remove from the global + New menu for now:
- New Project
- New Job
- New Invoice

3. Do NOT remove actual project/job/invoice creation from the system.
Only remove misleading global shortcuts that bypass workflow truth.

4. If useful, replace removed items with non-clickable contextual hints or omit them entirely.
Preferred: omit them entirely.

ACCEPTANCE CRITERIA
- + New only shows actions that are truly valid from anywhere in the app.
- No misleading workflow shortcuts remain in the global quick menu.

PHASE B — ESTIMATOR / QUOTE BUILDER DESKTOP POLISH
==================================================

OBJECTIVE
Turn the estimator into a stronger enterprise workspace rather than an older internal tool.

CURRENT ISSUE
The quote builder is functional but still visually inconsistent with the rest of the app:
- left config rail is dense and utilitarian
- drawing area is sparse
- item schedule is still visually secondary
- desktop hierarchy is not yet strong enough
- page looks like a separate product next to the improved tables/pages

REQUIRED IMPLEMENTATION
1. Quote items list default behavior
- On desktop, the quote items list must be visible by default.
- It must show a minimum of 3 rows/items worth of visible schedule space immediately.
- It must remain collapsible/expandable from its heading/toggle.
- The heading/toggle should expand to full list and collapse back to the default desktop height.
- On mobile/tablet, preserve practical responsive behavior.

2. Strengthen workspace hierarchy
The estimator page should clearly read as 3 coordinated zones:
- configuration rail (left)
- drawing / visual preview workspace (center)
- quote schedule / item list (bottom)

Improve spacing, borders, surface hierarchy, and vertical rhythm so these zones feel intentional and cohesive.

3. Improve quote items panel treatment
- Make the bottom quote-items surface feel like an operational schedule, not a secondary afterthought.
- Stronger heading, clearer collapsed/expanded affordance, better visual boundary from the drawing area.
- Maintain existing actions and controls.

4. Improve top action row
Review and tighten:
- Save
- Summary
- Exec Summary
- Download
- item count
Make this row feel more deliberate and enterprise-grade, not loose utility buttons.

5. Keep all current estimator functionality unchanged
Do not break:
- item add/edit/remove
- drawing preview
- summary / exec summary
- save
- download
- commercial extras
- pricing logic

ACCEPTANCE CRITERIA
- Estimator opens with quote schedule clearly visible on desktop.
- Minimum 3 visible item rows on desktop before expansion.
- User can expand/collapse from the heading.
- Workspace feels visually unified and closer to a premium commercial-operational tool.

PHASE C — LEGACY / UNLINKED INVOICE STATE CLARITY
==================================================

OBJECTIVE
Stop legacy broken records from appearing as normal clean records.

CURRENT ISSUE
The invoices list still shows rows with blank:
- customer
- project

while newer rows are correctly linked.
This creates silent ambiguity and undermines trust.

REQUIRED IMPLEMENTATION
1. Add explicit visual state for legacy/unlinked invoices
If an invoice lacks expected customer/project linkage, do NOT render silent blanks as if normal.
Instead show a clear state such as:
- Unlinked
- Legacy record
- Missing linkage

2. Differentiate between:
- truly missing live data
- legacy records created before workflow hardening
- incomplete records still in draft/recovery state

3. Add a repair path or management path for legacy/unlinked invoices
At minimum provide one of:
- “Manage in quote”
- “Repair linkage”
- “Review record”
depending on what is safest in current architecture

Do NOT invent unsafe auto-rewrite behavior.
Keep repair workflow controlled.

4. Keep valid invoices visually clean.
Do not overcomplicate correctly linked rows.

ACCEPTANCE CRITERIA
- Blank customer/project cells are no longer silently ambiguous.
- Legacy/unlinked invoices are visibly identified.
- There is a controlled path for review/repair.
