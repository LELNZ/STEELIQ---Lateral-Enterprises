1. DIVISION LOGO PERSISTENCE
Problem:
Division logo appears in settings temporarily but does not persist reliably in production. Broken image icon appears.

Diagnosis:
Current implementation uploads logos via /api/drawing-images using local disk storage (uploads/drawing-images) and stores only the URL/key in division_settings.logoUrl. This is not durable enough for production.

Required:
- Move division logo storage to durable persistence.
- Preferred:
  a. store logo binary in database-backed image storage already used elsewhere, or
  b. use durable object storage
- Save only a durable key/URL in division settings.
- Ensure the saved logo still renders after refresh/redeploy.
- Do not rely on local filesystem persistence for division logos.

Acceptance:
- Upload logo
- Save settings
- Refresh page
- Logo still displays
- Quote preview and PDF use the same persisted logo
