Corrections / Improvements to make before proceeding
1) The window.open signature

Your plan uses:
window.open(/quote/${quoteId}/preview, "_blank", "noreferrer")

In browsers, the 3rd arg is a features string, and "noreferrer" isn’t universally respected as a feature there. If you want to prevent referrer reliably, you typically do:

window.open(url, "_blank", "noopener,noreferrer")

So change to:

window.open(/quote/${quoteId}/preview, "_blank", "noopener,noreferrer");

This also avoids the “new tab can access opener” security issue.

2) Tooltip providers: don’t nest repeatedly if avoidable

The plan wraps just the disabled button with its own <TooltipProvider>. That’s fine, but if the app already uses a global provider (common in shadcn setups), adding another is unnecessary (still works though).

Best practice inside a page component:

Use a single <TooltipProvider> around the two buttons (or at least around that button group), then use <Tooltip> inside.

Not required, but cleaner.
